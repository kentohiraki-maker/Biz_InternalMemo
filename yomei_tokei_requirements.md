## カウントダウン表示仕様

- **表示単位**: 年 / 日 / 時間 / 分 / 秒を常時全単位同時表示
- **ミリ秒**: 廃止（チカチカするため削除）
- **計算式**: `(目標年齢の誕生日 - 現在時刻) × (1日の活動時間 / 24時間)`
- **活動時間**: 24時間から設定した睡眠時間を除いた時間

## 画面フロー（MVP）

1. **スプラッシュ / Google認証** — アプリ起動 → Googleサインイン → 購入状態チェック（RevenueCat）
2. **ペイウォール**（未購入時）— ぼかされたカウントダウン＋死神＋購入コピー＋CTA
3. **オンボーディング**（初回のみ）— 生年月日 → 目標年齢 → 起床/就寝時間の3ステップ
4. **メイン画面（カウントダウン）** — 全単位リアルタイム表示。砂時計アニメーション付き
5. **設定画面** — 目標年齢・起床/就寝時間の変更、通知設定、アカウント情報

## 技術スタック（MVP）

| カテゴリ | 技術 |
|----------|------|
| フレームワーク | React Native + Expo (managed) |
| 言語 | TypeScript |
| 認証 | Firebase Auth（Google Sign-In） |
| DB / 同期 | Firestore（設定・購入状態の複数端末同期） |
| 課金管理 | RevenueCat（買い切り ¥500 entitlement） |
| ローカル保存 | MMKV（オフライン高速アクセス） |
| 状態管理 | Zustand |
| 通知 | expo-notifications |
| iOSウィジェット | Swift + WidgetKit |
| Androidウィジェット | Kotlin + Jetpack Glance |
| ネイティブブリッジ | Expo Modules API |
| ナビゲーション | Expo Router |
| CI/CD | EAS Build + EAS Submit |

### 重要な実装メモ

- RevenueCat と Firebase Auth の連携: Firebase UID を RevenueCat の App User ID として使う
- ウィジェットのリアルタイム更新: iOSのWidgetKitはOS制約により秒単位更新＋アニメーション演出で対応
- Firebase Auth を使う場合、バックエンドとして Firebase (Auth + Firestore) が最速
